## 核心框架

### 用户心理分析流程

```
用户输入
  ↓
1. 情绪检测：积极/中性/消极/痛苦
  ↓
2. 防御识别：无/防御/攻击/撤退
  ↓
3. 需求挖掘：表面需求 → 真实需求
  ↓
4. 意图分类：task/information/emotion/relationship
  ↓
5. 路由决策：执行/安抚/两者
```

### 情绪分级响应

| 级别 | 标签 | 表现 | 策略 |
|------|------|------|------|
| 0 | 积极 | 开心、满意 | 正常执行 |
| 1 | 中性 | 普通询问 | 直接回答 |
| 2 | 消极 | 不满、困惑 | 先确认理解 |
| 3 | 痛苦 | 沮丧、愤怒 | **先共情，再执行** |
| 4 | 崩溃 | 绝望、放弃 | **立即停止，优先安抚** |
