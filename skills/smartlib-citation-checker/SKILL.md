# SmartLib 文献引用核查与纠错 / Citation Authenticity Checker

> 中文 / Chinese | [English below each section]

本技能基于 `smartlib-literature-search` 的文献检索能力，对用户提交的论文稿件或文本引用进行核查与纠错，最终输出一个结构化的 HTML 核查报告文件。

> This skill uses `smartlib-literature-search` to verify academic references and detect errors, generating a structured HTML report.

## 核心能力 / Core Capabilities

| 能力 / Capability | 说明 / Description |
|------|------|
| **引用解析 / Citation Parsing** | 自动识别并解析 GB/T 7714、APA、MLA、Chicago、BibTeX 等多种格式 / Auto-detect and parse GB/T 7714, APA, MLA, Chicago, BibTeX formats |
| **BibTeX支持 / BibTeX Support** | 完整解析 .bib 文件，支持 @article/@book/@inproceedings 等 entry 类型 / Parse .bib files with full entry type support |
| **真实性核查 / Authenticity Check** | 通过 SmartLib API 联网核查文献是否真实存在，**防止 AI 幻觉** / Verify references via SmartLib API, **detect AI hallucinations** |
| **信息比对 / Field Comparison** | 将原始引用与数据库记录逐字段对比，识别差异 / Compare original citations against database records field by field |
| **纠错输出 / Error Correction** | 对错误的作者、年份、期刊名、页码等信息提供修正版本（使用 `[删除]`/`[新增]` 标记）/ Provide corrected citations with `[删除]`/`[新增]` diff markers |
| **引用分析 / Citation Analytics** | 对批量文献进行年份分布、作者集中度、期刊集中度等统计分析 / Year distribution, author frequency, journal concentration analysis |
| **验证链接 / Verification Links** | 为每条匹配的文献提供可点击的验证链接 / Clickable verification links for each matched reference |
| **批量导出 / Batch Export** | 提供全量下载（Blob+createObjectURL，兼容所有浏览器环境）/ Blob-based download, compatible with all browser environments |
| **多格式输出 / Multi-format Output** | 保留原始格式，同时额外输出 GB/T 7714、APA、MLA、Chicago、BibTeX 五种常用格式 / Preserve original format, plus export to 5 standard formats |

## 适用场景 / Use Cases

**主要场景（核心价值）：**
- 🤖 **AI 幻觉验真**：用户使用豆包、ChatGPT、Kimi、文心一言等 AI 生成了参考文献，担心 AI "编造"了不存在的文献，提交核查
- 📄 **论文初稿审查**：作者对自己论文稿件中的参考文献进行真实性与格式规范性的自查，投稿前确保引用准确
- 📋 **粘贴式核查**：用户直接粘贴参考文献列表，快速批量验证和纠错

**其他场景（Other Scenarios）：**
- 提交 .bib 文件或粘贴 BibTeX 条目（Zotero/EndNote/LaTeX 用户）
- 编辑/审稿人批量核查来稿的引用准确性
- 对文献引用进行统计分析（年份分布、期刊偏好、作者集中度等）

> **English:**
> - 🤖 **AI hallucination detection**: Verify references generated by AI tools (Doubao, ChatGPT, Kimi, etc.) to ensure they actually exist
> - 📄 **Draft paper review**: Authors self-check citations before journal submission
> - 📋 **Paste-and-verify**: Quickly verify and correct a pasted reference list

**引用格式识别规则：**

```
GB/T 7714-2015 格式示例：
[1] 张三, 李四. 深度学习综述[J]. 计算机学报, 2024, 45(3): 123-145.
[2] Wang L, Li Y. Advances in machine learning[J]. Nature, 2023, 589: 45-52.

APA 格式示例：
Zhang, S., & Li, Y. (2024). Deep learning survey. 计算机学报, 45(3), 123-145.
Wang, L. (2023). Advances in machine learning. Nature, 589, 45-52.

MLA 格式示例：
Zhang, San, and Si Li. "Deep Learning Survey." 计算机学报, vol. 45, no. 3, 2024, pp. 123-145.
Chicago 格式示例：
Zhang, San, and Si Li. 2024. "Deep Learning Survey." 计算机学报 45 (3): 123-145.

BibTeX 格式示例：
@article{zhang2024deep,
  author = {Zhang, San and Li, Si},
  title = {Deep Learning Survey},
  journal = {计算机学报},
  year = {2024},
  volume = {45},
  number = {3},
  pages = {123--145}
}

纯文本格式（需智能识别）：
深度学习在医学影像中的应用. 张三, 李四. 计算机学报, 2024, 45(3): 123-145
```

**格式识别 → 表头标注规则**：

识别到原始文献的格式后，HTML 报告中的**两列**均需在表头标注格式名：
- **原始参考文献 (格式名)** — 如 `原始参考文献 (APA)`，让用户明确知道原始文献的格式
- **修正后文献 (格式名，与原始格式一致)** — 如 `修正后文献 (APA，与原始格式一致)`，右侧列头直接呼应左侧，一目了然

> 混排场景（多条文献各自不同格式）：原始列头标注为 `原始参考文献 (APA/GB-T-7714/...)`，修正列标注为 `修正后文献 (逐条与原文格式一致)`。

**BibTeX 解析规范：**

识别用户输入中的 `@article{`、`@book{`、`@inproceedings{`、`@mastersthesis{`、`@phdthesis{`、`@misc{` 等 entry 类型，提取以下字段：

| BibTeX 字段 | 对应标准字段 |
|-------------|------------|
| `author` | 作者列表（"and" 分隔） |
| `title` | 文献题名 |
| `journal` / `booktitle` | 期刊名 / 会议名称 |
| `year` | 出版年份 |
| `volume` | 卷号 |
| `number` | 期号 |
| `pages` | 页码范围 |
| `doi` | DOI |
| `url` | 原文链接 |
| `publisher` | 出版社 |
| `isbn` / `issn` | ISBN / ISSN |

解析策略：
1. 按 `@type{key,` 模式分割多条 entry
2. 逐字段提取 `field = {value}` 或 `field = "value"` 模式
3. `author` 字段支持 "and" 分隔多作者，自动标准化
4. `pages` 字段支持 "123-145" 和 "123--145"（双连字符）两种格式
5. 未找到 `journal` 时降级使用 `booktitle`

**解析策略：**

1. **逐行分割**：按换行或编号 `[1]` / `[1]` 等标记分割引用
2. **DOI优先提取**：优先匹配 DOI 模式 `10.xxxx/xxxx` 进行精确检索
3. **结构化提取**：
   - 英文文献：匹配 `Author(s). (Year). Title. Journal, Vol(Issue), Pages.` 模式
   - 中文文献：匹配 `作者. 题名[J]. 期刊名, 年份, 卷(期): 页码.` 模式
4. **模糊补全**：对格式不规范的文本，提取可识别的关键词（题名关键词、作者姓氏、年份等）

### Step 2: 联网核查 / Online Verification（调用 SmartLib API）

对每条解析后的引用，执行 SmartLib 文献检索核查。

> **⚡ 性能优化 / Performance Optimization（必须遵守）：**
>
> 本轮次的核心等待瓶颈在 API 网络调用。以下优化策略**必须严格遵循**以大幅缩短用户等待时间，不影响任何核查结果的准确性。
>
> #### 2.0 Token 缓存复用 / Token Caching
>
> - **启动时一次性获取** AccessToken，存入会话级变量，**全部文献共享同一 Token**
> - Access Token 有效期仅 30 秒（由 `tokenExpiredAt` 字段标示），Refresh Token 有效期 2 小时（`refreshTokenExpireAt`），远超单次核查耗时，同一批次内无需重复获取
> - 若某次 API 调用返回 Token 过期错误（HTTP 401 / `TokenExpired`），**仅此时**重新获取并刷新缓存
> - **缓存存储方式**：将 `token` 和 `tokenExpiredAt` 存入会话级临时变量；每次 API 调用前检查 `当前时间 < tokenExpiredAt`
> - **注意**：同一会话中多次核查（如用户追加文献）复用同一 Token，无需重复获取。Token 仅 30 秒有效，跨请求间隔较长时系统自动静默刷新，用户无感知。
> - **响应字段变更**：`data.token`（新）优先于旧版 `Data.AccessToken`；解析时优先使用小写 `data` 字段，兼容 `Data`。
>
> #### 2.1 并行检索 / Parallel Search
>
> - 将全部解析后的文献按 **每批 8 条** 分组，同批内 **并行发送** API 检索请求
> - **严禁串行逐条检索**（30条串行约需 60-120 秒，8 并发可降至 8-15 秒）
> - 并行检索示例（伪代码）：
>   ```
>   batch_size = 8
>   for batch in chunks(all_refs, batch_size):
>       results = await parallel([
>           search_ref(ref) for ref in batch
>       ])
>       merge(results)
>   ```
> - 每个并行请求独立处理错误：某条失败不影响同批其他文献
>
> #### 2.2 智能提前终止回退 / Early Fallback Termination
>
> - 优先级 1（题名+年份）返回 **≥ 3 条候选结果**时，**跳过**优先级 2 和 3
> - 优先级 1 返回 1-2 条但**题名相似度 > 80%** 时，同样跳过回退
> - 仅在优先级 1 **零结果**或候选结果**题名相似度均 < 50%** 时，才进入优先级 2
> - 优先级 3（DOI 校验）仅在优先级 2 也未找到、且原始引用中存在 DOI 时才执行
> - 此策略可将大部分文献的 API 调用次数从 **3 次降至 1 次**
>
> #### 2.3 进度反馈 / Progress Feedback
>
> - 在并行检索过程中，**向用户实时展示进度**：
>   ```
>   🔍 正在核查文献... [████████░░] 16/30 (当前批次: 第2批)
>   ```
> - 每完成一批（8 条）更新一次进度，给用户清晰的等待预期
> - 大幅减少"不知道在干什么"的等待焦虑感
>

>
> **检索优先级：**

| 优先级 | 检索方式 | 适用条件 |
|--------|---------|---------|
| 1 | 题名关键词 + 年份检索 | 题名可提取（所有文献） |
| 2 | 放宽年份检索（仅题名关键词） | 优先级1无结果时 |
| 3 | 提取DOI做格式校验 | DOI存在时作为辅助验证 |

> **注意：** SmartLib 全球文献检索 API 支持的 Rule 字段为 `T`(题名)、`A`(作者)、`K`(关键词)、`P`(出版物名称)、`O`(机构)、`U`(全部字段)。`Identifier_DOI` 不是有效检索字段，不可用于 Rule 表达式。

**检索表达式构建：**

```python
# 优先级1: 题名核心词 + 年份（去除a/an/the/in/of/for/and/on/to/is/by/with/using/based/from等停用词后取前6个词）
stopwords = {"a", "an", "the", "in", "of", "for", "and", "on", "to", "is", "by", "with", "using", "based", "from"}
words = [w for w in title.lower().replace(",", "").replace(":", "").split() if w not in stopwords][:6]
rule = f"(T={' '.join(words)}) AND Y={year}"

# 优先级2: 仅题名关键词（去掉年份限制，提高召回）
words_loose = [w for w in title.lower().replace(",", "").replace(":", "").split() if w not in stopwords][:5]
rule = f"T={' '.join(words_loose)}"

# 优先级3: DOI格式校验（仅验证DOI格式有效性，不用于API检索）
# 通过正则提取并验证 DOI 格式: 10.xxxx/xxxx
doi_match = re.search(r'10\.\d{4,}/[^\s]+', ref_text)
```

**接口选择：**

- 中文期刊/会议论文：优先使用接口1（中文期刊检索），补充接口4
- 英文论文/国际期刊：使用接口4（全球文献检索）
- 学位论文、专利、标准：使用接口4

**接口4（全球文献检索）可用检索字段：**

| 字段代码 | 含义 | 示例 |
|---------|------|------|
| `T` | 题名 / Title | `T=deep learning` |
| `A` | 作者 / Author | `A=Zhang S` |
| `K` | 主题词 / Keyword | `K=insulin` |
| `P` | 出版物名称 / Publication | `P=Nature` |
| `O` | 机构 / Organization | `O=Tsinghua University` |
| `U` | 全部字段 / All fields | `U=yeast insulin` |

> **注意**：`Identifier_DOI` 不是有效检索字段，不可用于 Rule 表达式。DOI 仅用于格式验证（正则匹配）和输出链接。

**检索表达式运算符：**

- 逻辑运算：`AND`（两边加空格）、`OR`、`NOT`
- 示例：`(T=yeast AND T=insulin) AND Y=2023`
- 示例：`(A=Zhang OR A=Wang) AND T=diabetes`

**检索参数：**

```json
{
  "Rule": "<检索表达式>",
  "PageIndex": 1,
  "PageSize": 10,
  "Sort": 1
}
```

### Step 3: 匹配与比对 / Match & Compare

对检索结果与原始引用进行智能匹配：

**匹配度计算：**

| 因素 | 权重 | 说明 |
|------|------|------|
| 题名相似度 | 60% | 使用共同词比例算法（Jaccard-like），>50%认为匹配 |
| 作者匹配 | 25% | 至少一位作者匹配，且顺序基本一致 |
| 年份匹配 | 15% | 年份完全一致 |
| DOI辅助验证 | 加分项 | DOI匹配可增加置信度，但不作为主要检索方式 |

**匹配结果分类：**

| 结果 | 状态码 | 说明 | 验证链接 |
|------|--------|------|---------|
| ✅ 验证通过 | VERIFIED | 找到高度匹配的文献，信息准确 | 提供 |
| ⚠️ 存在差异 | MISMATCH | 找到匹配文献，但部分字段有差异 | 提供 |
| ❌ 未找到 | NOT_FOUND | 未在数据库中找到匹配的文献 | 不提供 |
| ❓ 疑似匹配 | FUZZY_MATCH | 找到疑似结果，需人工确认 | 不提供（标注存疑） |

### Step 4: 差异分析 / Diff Analysis（针对 MISMATCH）

逐字段对比原始引用与数据库记录：

| 字段对比 | 差异类型 | 纠错建议 |
|---------|---------|---------|
| 作者姓名 | 拼写错误、缺少中间名、顺序错误 | 提供正确作者列表 |
| 年份 | 年份错误（早于实际发表年） | 提供正确年份 |
| 期刊名 | 缩写不规范、全称/简称混淆 | 提供标准期刊名 |
| 卷期号 | 卷号/期号错误 | 提供正确卷期 |
| 页码 | 起止页码错误 | 提供正确页码范围 |
| 题名 | 打字错误、遗漏词汇 | 提供正确题名 |
| DOI | DOI错误或缺失 | 提供正确DOI |

**差异识别算法：**

1. **作者姓名标准化**：去除空格、转换大小写后比较
2. **年份比较**：整数比较，支持"2024年" vs "2024"
3. **期刊名匹配**：支持缩写与全称匹配（如 "Nature" vs "Nature Publishing Group"）
4. **页码验证**：检查页码范围合理性（如 500-100 vs 100-500）

## 注意事项 / Notes

- **题名检索优先**：SmartLib 全球检索 API 不支持 `Identifier_DOI` 字段，请使用题名核心词 + 年份构建 Rule 进行检索。DOI 仅用于格式验证（正则匹配 10.xxxx/xxxx）和输出链接
- **中文期刊**：优先使用接口1（含全文下载链接），英文文献用接口4
- **批量处理**：单次核查不超过50条，超出时分批处理
- **BibTeX支持**：完整解析 .bib 格式，支持 `and` 分隔多作者、`--` 双连字符页码，自动映射 journal/booktitle 字段
- **多格式输出**：核查结果同时输出 GB/T 7714、APA、MLA、Chicago、BibTeX 五种格式，保留原始格式不动，各格式附带使用场景标签。**修正后主输出始终与原始输入格式一致**，若所有文献格式统一则从 Tab 中排除该格式
- **格式保持原则**：修正后文献输出格式 = 原始输入格式。逐条识别并保持一致，不改变用户原始引用的格式体系。**报告表格两列表头均标注格式名**：`原始参考文献 (APA)` + `修正后文献 (APA，与原始格式一致)`，双列呼应，让用户明确原始格式与修正列的关系
- **FUZZY_MATCH处理原则**：不瞎编修正内容，保留原始引用并明确标注存疑，要求用户手动核实
- **隐私保护**：用户上传的内容仅用于本次核查，不做存储
- **验证链接格式**：`https://data.smart.vipslib.com/articlesearch/web_searchingDetail?id={文献Identifier}`
- **统计分析触发条件**：文献数量 >= 3 时自动输出，少于3条时跳过
- **输出格式**：HTML报告文件，差异内容使用 `[删除]`/`[新增]` 文字标记（CSS样式区分颜色）
- **下载兼容性**：使用 Blob+createObjectURL 实现，兼容 file:// / http:// / https:// 所有环境

### Step 2 — 联网检索
